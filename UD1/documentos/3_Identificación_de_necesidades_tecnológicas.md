## **3. Identificación de necesidades tecnológicas**

Tras analizar el tipo de empresa objetivo, se detectan varias necesidades claras:

- **Automatizar auditorías de seguridad** sin intervención humana.  
- **Escanear vulnerabilidades web** (puertos abiertos, headers inseguros, SSL...).  
- **Auditar WordPress específicamente**, ya que es el CMS más usado y el más atacado.  
- Contar con **informes simples y entendibles** para usuarios no técnicos.  
- Recibir **alertas automáticas** si aparece una vulnerabilidad crítica.  
- Disponer de **un panel web fácil de usar**, accesible desde cualquier dispositivo.  
- Herramienta **económica, ligera y adaptable**, sin instalaciones complicadas.

Estas necesidades justifican totalmente el desarrollo de una plataforma automatizada.
