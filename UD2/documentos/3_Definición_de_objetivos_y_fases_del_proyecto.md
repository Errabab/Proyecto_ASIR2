## **3. Definición de objetivos y fases del proyecto**

### ✅ **Objetivo general**
Desarrollar una plataforma automatizada que realice auditorías de seguridad web orientadas a PYMES, incluyendo un módulo específico para WordPress y un panel de control intuitivo.

---

### ✅ **Objetivos específicos**

- Automatizar escaneos web comunes.  
- Detectar vulnerabilidades y configuraciones inseguras.  
- Analizar WordPress con herramientas especializadas.  
- Crear un panel web comprensible para usuarios no técnicos.  
- Implementar alertas automáticas por email/Telegram.  
- Generar informes claros y profesionales.  
- Permitir auditorías periódicas sin intervención manual.  
- Desplegar el sistema mediante contenedores Docker.

---

### ✅ **Fases del proyecto**

1. **Investigación y recopilación técnica.**  
2. **Desarrollo del módulo de auditoría web general.**  
3. **Desarrollo del módulo especializado para WordPress.**  
4. **Integración de herramientas (Nmap, ZAP, WPScan…).**  
5. **Diseño y creación del panel web.**  
6. **Montaje de base de datos.**  
7. **Sistema de alertas.**  
8. **Automatización de tareas.**  
9. **Pruebas técnicas y corrección de errores.**  
10. **Documentación final del proyecto.**
