## **1. Procedimientos de seguimiento**

Durante el desarrollo del proyecto he aplicado un seguimiento continuo para asegurar que cada fase avanzaba correctamente.  

Los procedimientos principales de seguimiento han sido:

### ✅ **Control de versiones con Git**
Todo el código y los cambios se han gestionado con GitHub, lo que permite:
- Revisar el progreso de cada módulo.
- Mantener un historial claro de modificaciones.
- Asegurar que no se pierde trabajo y poder volver a versiones estables.

### ✅ **Revisiones semanales**
Cada semana he revisado:
- Estado de los módulos (auditoría general, WordPress, panel web…).
- Errores encontrados.
- Objetivos cumplidos vs. pendientes.
- Prioridades para la semana siguiente.

### ✅ **Pruebas continuas en AWS**
Al usar AWS educativo como entorno principal:
- Pude comprobar en tiempo real si los contenedores levantaban bien.
- Se verificó el funcionamiento de MariaDB, Docker y los scripts.
- Se validaron auditorías reales en sitios de prueba.

### ✅ **Checklist por fases**
Para cada etapa he mantenido una lista de tareas completadas, lo que permite llevar un control más visual y sencillo del progreso.
